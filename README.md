# Dumb & Becket Design

> *"L'innovation n'est pas nécessairement synonyme d'utilisabilité"*

## Le projet

En 2017, une équipe de designers s'est amusée à créer [les pires interfaces de contrôle du volume possibles](https://uxdesign.cc/the-worst-volume-control-ui-in-the-world-60713dc86950). Le résultat ? Un exercice de style brillant qui démontre par l'absurde les fondamentaux du design d'interface.

Comme le souligne l'article original : *"Sometimes the best way to learn what good design looks like is to see examples of really, really bad design."* Ces interfaces volontairement horribles nous rappellent que chaque décision de design - le placement d'un bouton, le comportement d'un slider, le feedback visuel - a un impact direct sur l'expérience utilisateur.

À l'heure où n'importe qui peut générer une application en quelques prompts, ce message n'a jamais été aussi pertinent.

## L'objectif

Ce repo a pour vocation de **fédérer les efforts des designers, aspirants développeurs, vibe-coders et curieux** dans une recherche collective de l'interface **la moins utilisable possible**.

Parce que l'URSSAF n'a pas le monopole des mauvaises interfaces.

## Les contributions actuelles

### `patience.html` - La barre de chargement de la frustration

Une barre de progression conçue pour maximiser l'irritation :

- **Paradoxe de Zénon** : Plus on approche de 100%, plus la progression ralentit exponentiellement
- **Régression aléatoire** : La barre peut reculer sans prévenir ("Whoops, dropped a packet...")
- **Bouton Cancel fugitif** : Le bouton s'enfuit quand on essaie de cliquer dessus
- **Messages passifs-agressifs** : "Still waiting...", "Almost there (not really)..."
- **Esthétique Windows 95** : Comic Sans, bleu électrique, et bordures biseautées

## Comment contribuer

1. Fork ce repo
2. Créez votre interface inutilisable (HTML, CSS, JS, ou tout autre format)
3. Documentez les "anti-patterns" UX que vous exploitez
4. Soumettez une PR avec une description de votre chef-d'oeuvre

### Critères d'excellence (dans la médiocrité)

- **Frustration maximale** : L'utilisateur doit douter de ses propres capacités
- **Créativité** : Trouvez de nouvelles façons de torturer l'UX
- **Humour** : Le rire est la meilleure défense contre la mauvaise UI
- **Éducatif** : Chaque mauvaise décision doit illustrer un bon principe par contraste
